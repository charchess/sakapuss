## Overview

**Page Purpose:** Complete medical context in 30 seconds -- the whole point of the vet portal. Dr. Martin sees proactive alerts, key health facts, weight trends, and a filtered medical timeline without Camille having to explain anything.

**User Situation:** Dr. Martin clicked on Pixel's card or search result. He's in consultation with Camille sitting across from him. Every second he spends navigating is visible to her. He needs to absorb Pixel's health context at a glance and know what questions to ask.

**Success Criteria:**
- Full context absorbed in < 30 seconds
- Alerts are PROACTIVE -- Dr. Martin sees signals before speaking
- Weight chart visible without scrolling
- Medical timeline filtered to relevant categories by default
- Toggle to show all data (including Activity) available but not default

**Entry Points:**
- Search result from 08.1
- Patient card click from 08.1
- Alert row click from 08.1 (scrolls to alerts)
- Direct URL (bookmarked patient)

**Exit Points:**
- Back to dashboard -> 08.1
- Search (top bar) -> new search from any page
- Next/prev patient (if navigating from list)

## Technical Notes

- **30-second principle:** The layout is designed so that without ANY interaction, Dr. Martin can absorb: (1) alerts at top, (2) key facts in left column, (3) weight trend in chart, (4) recent medical history in right column. All above the fold on a 1080p monitor.
- **Alert computation:** Server-side. Weight trend: linear regression over last 3 data points. Overdue: reminder date vs. today. Behavioral: frequency counting within sliding windows.
- **Data quality filtering:** Default "Medical" filter hides Activite category which contains high-frequency low-signal entries (playing, sleeping). Vet can toggle "Show all" to see everything.
- **Weight chart:** SVG-based, lightweight. Renders from raw weight entries. No external charting library dependency (or use a minimal one).
- **Performance:** Full dossier data should load in < 1 second. Timeline uses virtual scrolling for patients with 100+ entries.
- **Print:** Add print stylesheet that formats the page as a clean, single-column report. Header + alerts + quick facts + weight chart + timeline.
- **Responsive:** Desktop primary. On tablet, columns stack vertically (summary above timeline). Search bar remains in top bar.
- **Read-only:** This is a read-only view. Dr. Martin cannot edit, add, or delete entries. This is the owner's data.
