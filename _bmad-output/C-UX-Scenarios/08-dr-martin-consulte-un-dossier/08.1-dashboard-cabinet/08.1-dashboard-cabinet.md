## Overview

**Page Purpose:** Dr. Martin's practice portal landing page -- search for a patient instantly and access recent dossiers. Every second counts during consultation.

**User Situation:** Dr. Martin is in consultation with Camille. He opens his browser bookmark for Sakapuss portal, or he's already logged in. He needs to find Pixel's dossier in under 5 seconds.

**Success Criteria:**
- Search bar is the first focusable element (auto-focused on page load)
- Patient found in < 3 keystrokes (type-ahead search)
- Recent patients accessible in 1 click
- Alert badges surface critical signals before opening dossier

**Entry Points:**
- Browser bookmark (primary)
- Portal login -> redirects here
- After portal creation (07.2) -> redirects here

**Exit Points:**
- Search/click patient -> 08.2 Dossier patient
- Settings -> portal settings
- Log out

| State | When | Appearance | Actions |
|-------|------|------------|---------|

| Search Active | Typing in search | Dropdown with matching results | Select result |
| Search No Results | Query matches nothing | Dropdown with "No patients found" message | Modify query |
| Loading | Page loading / data sync | Skeleton: search bar + 4 card placeholders + 3 table row placeholders | Wait |

## Technical Notes

- **Auto-focus search:** On page load, cursor is in search bar. This is the #1 action -- finding a patient.
- **Type-ahead:** Search queries server after 1 character with 200ms debounce. Results ranked by: exact name match > partial name > owner name.
- **Keyboard navigation:** Full keyboard support for search results (arrow keys, enter, escape). Essential for fast clinic workflows.
- **Recent patients:** Based on last viewed, not last shared. Max 8 cards, horizontal scroll.
- **Alert aggregation:** Alerts are computed server-side from statistical anomalies (weight trends, behavior patterns) and overdue reminders. Refreshed on page load.
- **Performance:** Dashboard must load in < 500ms. Search results in < 200ms. No pagination on search results (max 20 results shown).
- **Session:** Long-lived session (30 days). Browser remember-me. Clinic computers are shared -- offer "Log out" prominently.
- **Responsive:** Desktop-first but usable on tablet. Below 768px, recent cards stack vertically, table becomes card list.
