## Overview

**Page Purpose:** Camille generates a permanent revocable access link for her veterinarian and sends it by email. The link gives immediate read-only access to the animal's health dossier -- no account required for the vet to view it.

**User Situation:** Camille has a vet appointment in 3 days. She wants Dr. Martin to arrive prepared with Pixel's full health context. She navigates to sharing, selects Pixel, enters Dr. Martin's email, and sends the link.

**Success Criteria:**
- Share flow completed in < 60 seconds
- Link is permanent (not expiring) but revocable by Camille at any time
- Camille understands what the vet will see before sending
- Existing shares are manageable (view, revoke)

**Entry Points:**
- Animal profile -> "Share with my vet" CTA
- Settings/menu -> "Vet sharing" section
- Timeline -> contextual "Share this with your vet?" prompt (after health entries)

**Exit Points:**
- Email sent with link -> return to animal profile or share management
- Back arrow -> previous page

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default (no shares) | First visit, no existing shares | Empty active shares + new share form | Create first share |
| Default (with shares) | Has existing shares | Active share cards + new share form | Manage shares or create new |
| Form Valid | Animal selected + valid email | Send button enabled | Send |
| Sending | Send button tapped | Button shows spinner | Wait |
| Link Sent | API confirms delivery | Success toast: "Link sent to {email}!" New card in active shares. | Return to management |
| Revoke Confirm | Revoke tapped | Confirmation dialog: "Revoke access for {vet_email}? They will no longer be able to view the dossier." | Confirm or cancel |
| Revoked | Confirmed revoke | Card status changes to "Revoked", then fades out after 3s | -- |
| Send Error | Email delivery fails or API error | Inline error below form | Retry |
| Offline | No network | Active shares from cache, send disabled | View only |

## Technical Notes

- **Link model:** Permanent URL containing encrypted token. Token maps to: household ID, animal IDs, creation date, revocation status. No expiry by default.
- **Revocation:** Instant. Server-side flag. Link URL still resolves but shows "Access revoked" message.
- **Email content:** Email includes: Camille's name, animal names, direct link, brief explanation of Sakapuss. Designed for Dr. Martin to understand without prior context.
- **Duplicate prevention:** If Camille tries to share with an email that already has an active link for the same animals, show warning: "This vet already has access. Send a reminder instead?"
- **Link security:** Token is cryptographically signed. Rate-limited access. Vet IP logged for audit.
- **Copy link:** Long-press on active share card shows "Copy link" option for sharing via other channels.
