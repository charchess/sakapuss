## Overview

**Page Purpose:** Dr. Martin opens Camille's link and sees the patient dossier immediately -- zero friction, no account required. A non-blocking banner offers optional practice portal creation for centralizing all Sakapuss patients.

**User Situation:** Dr. Martin received an email from Sakapuss on behalf of Camille. He's at his cabinet computer, has 2 minutes before the appointment. He clicks the link to see Pixel's health context.

**Access Model:**
- **Without account:** Link opens -> dossier visible immediately (read-only)
- **With new account:** "Create my practice portal" -> minimal account creation -> Pixel auto-linked
- **With existing account:** "Add to my portal" -> Pixel linked to existing portal

**Success Criteria:**
- Dossier visible in < 2 seconds after clicking link (no gates, no walls)
- Portal creation CTA is visible but never blocks the dossier
- If Dr. Martin creates an account, it takes < 60 seconds
- Pixel auto-links to portal without manual steps

**Entry Points:**
- Email link from Camille (primary)
- Direct URL (bookmarked or shared)

**Exit Points:**
- View dossier and close (no account created)
- Create portal -> 08.1 Dashboard cabinet
- "Add to my portal" (existing account) -> 08.1 Dashboard cabinet with Pixel added

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default (anonymous) | Valid link, no account | Dossier visible + banner + top bar CTA | View dossier, optionally create portal |
| Default (logged in) | Valid link, has portal account | Dossier visible + "Add to my portal" CTA | View dossier, add patient |
| Banner Dismissed | User closed banner | Dossier visible, no banner, top bar CTA remains | View dossier |
| Modal Open | CTA tapped | Portal creation modal overlays | Fill form |
| Creating Account | Submit in progress | Button spinner | Wait |
| Account Created | Success | Modal closes, redirect to 08.1 with Pixel linked | Auto-navigate |
| Patient Added | Existing user, "Add to portal" | Toast: "Pixel added to your portal" + redirect to 08.1 | Auto-navigate |
| Revoked Link | Camille revoked access | Error page: "This access has been revoked by the pet owner." | Close page |
| Invalid Link | Malformed token | Error: "This link is invalid." | Close page |
| Loading | Page loading | Skeleton dossier (header + 3 timeline placeholders) | Wait |

## Technical Notes

- **Zero-friction principle:** The dossier MUST be visible without any authentication. Portal creation is an upgrade, never a gate.
- **Link resolution:** Server validates token, returns patient data as JSON. Page renders client-side. No redirects, no interstitials.
- **Auto-linking:** When Dr. Martin creates a portal account, the patient from the link is automatically added to his patient list. No manual steps.
- **Existing account detection:** If the email entered in the portal form already has an account, offer to log in instead. Don't create a duplicate.
- **Desktop-first:** This page targets cabinet computers. Desktop layout is primary. Mobile responsive as secondary.
- **SEO/Privacy:** Page must NOT be indexable. robots.txt noindex. Token-only access.
- **Performance:** Dossier data should load in < 1 second. Skeleton loading for timeline entries.
- **Banner persistence:** Dismissed state stored in localStorage per link. If Dr. Martin visits another patient link, banner reappears.
