## Overview

**Page Purpose:** Complete animal card — identity, health data, active reminders, recent activity, and the gateway to vet sharing. This is the single source of truth for everything about one animal. When Camille is at the vet and needs to answer questions, this is the page she opens.

**User Situation:** Camille taps Pixel's avatar from anywhere in the app. She may be browsing casually, or she may be at the vet's office needing to share information. This page gives her confidence: everything about Pixel is here, organized, accessible.

**Success Criteria:**
- All essential animal info visible without scrolling (identity card)
- Active reminders and weight trend visible with one scroll
- "Share with my vet" CTA prominent — this is where Camille decides to share
- Edit access clear and easy

**Entry Points:**
- Animal avatar tap from Dashboard (primary)
- Animal avatar tap from Timeline
- Animal avatar tap from Reminder detail
- Bottom nav "Profile" tab → animal selection
- Any animal avatar tap throughout the app

**Exit Points:**
- "Share with my vet" → 07.1 Share flow
- Tap weight section → 04.2 Graphique de poids
- Tap reminder card → 03.2 Rappel detail
- Tap "See full history" → 04.1 Timeline (filtered to this animal)
- Edit button → Edit animal flow (01.2 variant)
- Back → previous page

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Animal with data | Full profile: identity + weight + reminders + activity + share | All actions |
| New Animal | Just added, minimal data | Identity card filled, weight/reminders/activity sections show empty states with CTAs | Log first weight, set reminder |
| No Weight | No weight entries logged | Weight section: "No weight logged yet" + "Weigh now" CTA | Quick Log (weight) |
| No Reminders | No active reminders | Reminders section: "No active reminders" + "Set up" CTA | Create reminder |
| No Vet | No vet linked | Vet field: "No vet linked" + "Link a vet" CTA | Link vet flow |
| Shared with Vet | Vet has access to this animal | Share button changes to "Shared with Dr. Martin ✓" | Manage sharing settings |
| Loading | Fetching animal data | Skeleton identity card + skeleton sections | Wait |
| Offline | No network | Cached data shown, offline indicator | View only, actions queued |

## Technical Notes

- **Universal entry point:** This page is reachable from any animal avatar tap in the app. The back button always returns to the previous context.
- **Data aggregation:** This page aggregates data from multiple sources: animal identity (profile), weight entries (logs), reminders (scheduler), activity (timeline). All cached locally for offline access.
- **Age calculation:** Dynamic from DOB. Shows years + months for < 2 years (e.g., "1 year 4 months" / "1 an 4 mois"), years only for older animals.
- **Weight trend:** Same calculation as 04.2, but displayed as a compact summary. Tap through to full chart for details.
- **Share state persistence:** Once Camille shares with a vet, the button state updates globally. The "Shared" state shows on every visit.
- **Edit permissions:** Only household admin can edit animal details. Other household members see the profile read-only without the edit icon.
- **Chip number format:** Validated as 15-digit ISO standard. Displayed with no separators. Copy-to-clipboard on long press.
- **Photo:** Stored locally + synced to server. Max 5MB. Crop to square on upload. Fallback: species emoji + first letter on `color-primary-light`.
