## Overview

**Page Purpose:** Consolidated view of all reminders segmented by status — today, upcoming, and overdue. The reminder that needs immediate attention surfaces at the top. This is the command center for Camille's animal care rhythm.

**User Situation:** Camille just gave Pixel his dewormer. She taps the push notification "Vermifuge de Pixel — aujourd'hui" and lands here. She sees the reminder highlighted as today's priority. One more tap to validate.

**Success Criteria:**
- Today's reminder visible without scrolling (first position)
- Overdue count visible immediately via badge — nothing disappears silently
- Time to reach detail/validation: 1 tap from this page

**Entry Points:**
- Push notification tap (primary — deep link to this page with today's segment focused)
- Bottom navigation "Reminders" tab
- Dashboard reminder banner "View all" link

**Exit Points:**
- Tap any reminder card → 03.2 Rappel détail + validation
- Bottom nav → Home, Timeline, Profile

## Technical Notes

- **Reminder lifecycle rule:** An unvalidated reminder NEVER disappears silently. Overdue reminders escalate visually but remain visible until explicitly validated or postponed.
- **J-7 notification flow:** Reminders appearing in "Upcoming" within 7 days show actionable CTAs (order product / book appointment) per the anticipation notification design.
- **Sort order within segments:** Overdue sorted by most overdue first (oldest). Today sorted by creation time. Upcoming sorted by nearest due date first.
- **Pull-to-refresh:** Sync reminder states from server.
- **Push notification deep link:** When user taps push notification, land on this page with the relevant reminder card briefly highlighted (pulse animation, `color-primary-light` bg flash).
- **Multi-animal:** Reminders for all animals appear together, distinguished by animal avatar on each card. No per-animal filtering on this page (use animal profile for that).
