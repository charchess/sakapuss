## Overview

**Page Purpose:** Full reminder context with Done/Postpone actions. This is the MOMENT DE VERITE — the first complete reminder cycle closes here. Camille sees the care she just performed validated, and the next occurrence auto-planned. The cycle closes, serenity begins.

**User Situation:** Camille just gave Pixel his dewormer. She is on this page to validate the action. She wants to confirm "done" and see when the next one is due — proof the system works for her.

**Success Criteria:**
- "Done" action reachable in 1 tap, thumb zone
- Success state clearly communicates: what was validated + when the next one is planned
- Postpone options visible but secondary — not competing with Done
- Time from page load to validation complete: < 3 seconds

**Entry Points:**
- Tap reminder card from 03.1 Liste des rappels (primary)
- Push notification deep link (direct to this page with `reminder_id`)

**Exit Points:**
- "Done" → success state → auto-return to 03.1 after 2s or tap to dismiss
- "Postpone" → confirmation → return to 03.1 with updated date
- Back navigation → 03.1 Liste des rappels

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default — Today | Reminder is due today | Green status, full context, Done prominent | Done, Postpone |
| Overdue | Reminder is past due | Red status indicator on date, urgent tone | Done, Postpone |
| Upcoming (J-7) | Reminder is within 7 days | Yellow status, anticipation CTAs visible (order/book) | Done (disabled), Postpone, Anticipation actions |
| Postpone Expanded | User tapped Postpone | 3 option chips visible below Postpone button | Select delay option |
| Success | User tapped Done | Sunshine animation + completion message + next date | Auto-dismiss or tap back |
| Loading | Fetching reminder data | Skeleton layout matching info card structure | Wait |
| Error | Failed to save validation | Error toast: "Could not save. Tap to retry." | Retry |
| Offline | No network | Show cached data, Done action queued for sync | Done (queued), visual offline indicator |

## Technical Notes

- **Optimistic update:** "Done" immediately shows success state, API call happens in background. If API fails, show error toast with retry.
- **Next occurrence auto-calculation:** Server calculates next reminder date based on frequency. Success state shows the calculated date immediately.
- **Postpone is not snooze:** Postpone moves the actual due date. The original date is stored in history for audit. An unvalidated reminder NEVER disappears — postponing just reschedules it.
- **Deep link support:** Push notification tap opens directly to this page with `reminder_id` param. If reminder already validated, show the success state (read-only).
- **Offline queue:** If Done is tapped offline, store locally and sync when reconnected. Show queued indicator on the reminder in 03.1.
- **Haptic feedback:** iOS and Android haptic on Done tap for tactile confirmation.
- **Accessibility:** Done button has `aria-label` "Mark {reminder_type} for {animal_name} as done". Success announcement via `aria-live="assertive"`.
