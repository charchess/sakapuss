## Overview

**Page Purpose:** Returning user authentication. Simple: email + password, social auth, forgot password. Mirrors the inscription page layout for visual consistency.

**User Situation:** Camille reopens the app or returns after session expiry. She wants to get back to her Dashboard as fast as possible. No patience for friction.

**Success Criteria:**
- Login completed in < 10 seconds
- Social auth available for one-tap return
- Forgot password accessible without confusion
- Clear path to signup for new users who land here by mistake

**Entry Points:**
- App open (session expired or first open after install)
- "Already have an account" from 01.1 Inscription
- Deep link requiring authentication

**Exit Points:**
- Successful login → 02.1 Dashboard
- "Create an account" → 01.1 Inscription
- "Forgot password" → Password reset flow (inline)

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Page load | Empty form, CTA disabled | Fill form or social auth |
| Form Valid | Both fields have input | CTA enabled | Submit |
| Loading | Submitted | Spinner on CTA, inputs disabled | Wait |
| Error — Wrong Credentials | Invalid email/password | Inline error: "Incorrect email or password" / "Email ou mot de passe incorrect" | Retry |
| Error — Network | No connectivity | Toast error message | Retry |
| Forgot — Email Prompt | Tapped "Forgot password" | Password field replaced by "Send reset link" button + helper text | Enter email + submit |
| Forgot — Sent | Reset email sent | Success message: "Check your email for a reset link" / "Vérifiez votre email pour un lien de réinitialisation" | Return to login form |
| Biometric Prompt | Device supports biometric + user opted in previously | Biometric sheet overlays on load | Authenticate or dismiss to show form |

## Technical Notes

- **Session persistence:** JWT stored in secure storage. Auto-login on app reopen if token valid.
- **Biometric auth:** If device supports Face ID / fingerprint and user opted in from Settings, offer biometric on load before showing the form.
- **Rate limiting:** After 5 failed attempts, show captcha or cooldown timer.
- **Forgot password:** Uses email-based reset link. No SMS for v1. Reset flow happens inline (no separate page).
- **Keyboard:** Email field auto-focused. "Next" moves to password. "Go" submits.
- **Deep link handling:** If user tapped a deep link requiring auth, store the destination and redirect after successful login.
