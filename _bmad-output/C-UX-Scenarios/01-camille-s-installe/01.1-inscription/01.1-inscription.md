## Overview

**Page Purpose:** Account creation — the very first interaction with Sakapuss. Maximum simplicity: email + password or social auth. No friction, no email verification blocker. Camille creates her account and moves immediately to adding her animals.

**User Situation:** Camille la Gardienne arrives one evening at home, phone in hand. She found Sakapuss via a vet link, app store, or web search ("app suivi santé chat"). Motivation is fresh but patience is limited — she will abandon if it takes more than 30 seconds.

**Success Criteria:**
- Account creation completed in < 30 seconds
- No email verification blocker before proceeding
- Clear path to next step (add animals) immediately after signup
- Social auth available as faster alternative

**Entry Points:**
- Vet link (pre-filled vet association, if applicable)
- App store / PWA install prompt
- Web search landing → signup CTA
- "Create account" link from 01.4 Login page

**Exit Points:**
- Successful signup → 01.2 Ajout animal
- "Already have an account" → 01.4 Login

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Page load, empty form | All inputs empty, CTA disabled | Fill form or tap social auth |
| Form Valid | Both fields pass validation | CTA enabled (`color-primary`) | Submit form |
| Loading | Form submitted, awaiting response | CTA shows spinner, inputs disabled | Wait |
| Error — Email Taken | Server returns duplicate email | Inline error below email field: "This email is already registered" / "Cet email est déjà utilisé" + link to login | Fix email or go to login |
| Error — Network | No connectivity | Toast: "No connection. Please try again." / "Pas de connexion. Veuillez réessayer." | Retry |
| Error — Server | 500 response | Toast: "Something went wrong. Please try again." / "Une erreur est survenue. Veuillez réessayer." | Retry |
| Vet Link Pre-fill | Arrived via vet invitation link | Vet name shown: "Invited by Dr. Martin" / "Invité(e) par Dr. Martin" — vet association stored silently | Normal signup flow |

## Technical Notes

- **No email verification blocker.** Account is created and usable immediately. Verification email sent in background — gentle reminder later (not during onboarding).
- **Vet link handling:** If URL contains `?vet=<vet_id>`, store the vet association on account creation. Show "Invited by {vet_name}" badge in the brand section.
- **Password requirements:** Minimum 8 characters. Strength indicator is advisory only — does not block submission.
- **Social auth:** Google and Apple only for v1. OAuth flow returns to 01.2 Ajout animal on success.
- **Auto-login:** After account creation, user is immediately authenticated — no login step required.
- **PWA:** Page works offline for viewing but requires connectivity for account creation. Show clear network error if offline.
- **Keyboard:** First input auto-focused on mobile. "Next" keyboard button moves to password. "Go" submits form.
- **Analytics:** Track signup method (email vs Google vs Apple), time-to-complete, vet link attribution.
