## Overview

**Page Purpose:** Define the user's animals — name and species are mandatory, everything else is optional. Camille adds Pixel (cat) then Luna (cat) using the "Add another" loop. Photo is optional. The page prioritizes speed: get animals into the system fast so the user feels investment.

**User Situation:** Camille just created her account. She is engaged and ready to set up. This is the moment of highest motivation — capitalize on it without overwhelming her.

**Success Criteria:**
- At least 1 animal added in < 30 seconds
- "Add another" loop is obvious and frictionless
- Optional fields do not slow down the mandatory path
- Species selection is visual and fast (icons, not dropdowns)

**Entry Points:**
- Successful signup → automatic redirect from 01.1
- "Add animal" from Dashboard empty state
- "Add animal" from Profile/Settings

**Exit Points:**
- "Continue" (after at least 1 animal) → 01.3 Onboarding Admin
- Skip not available — at least 1 animal is required for the app to function

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default — First animal | Fresh from signup, no animals | Empty form, "Continue" disabled, no animal list | Fill form |
| Form Valid | Name + species filled | Both action buttons enabled | Save or continue |
| Form Partial | Only name or only species | Buttons disabled | Complete required fields |
| Animal Saved — Add Another | 1+ animals saved, form reset | Empty form + animal list visible below. Celebration micro-animation on chip appear | Fill next animal |
| Animals Exist — Empty Form | Animals saved, form intentionally empty | "Continue" enabled (no save needed), "Save & add another" hidden | Continue to 01.3 |
| Loading | Saving animal | Button shows spinner | Wait |
| Error — Duplicate Name | Same name as existing animal | Inline warning: "You already have an animal named {name}" / "Vous avez déjà un animal nommé {name}" | Change name or confirm |
| Photo Uploading | User selected a photo | Photo circle shows upload progress ring | Wait |

## Technical Notes

- **Minimum 1 animal required.** The app's core loop (Quick Log) needs at least one animal to function. "Continue" button is only active when at least 1 animal exists.
- **Photo upload:** Resize client-side to max 512px before upload. Store thumbnail for avatar. Upload async — do not block form submission.
- **Breed autocomplete:** Breed database filtered by selected species. Supports free text entry if breed not in list.
- **Vet link pre-fill:** If user arrived via vet invitation link (01.1), the vet field is pre-populated and shown in the optional section (expanded by default in that case).
- **"Other" species:** When selected, shows a text input for custom species name. Onboarding admin (01.3) will offer generic reminders only.
- **Form reset animation:** After "Save & add another", form fields slide out left, new empty form slides in from right. Chip appears with a pop animation.
- **Data persistence:** Animals save individually to the backend. If user abandons mid-flow, previously saved animals persist.
- **Offline:** Form can be filled offline. Save queued for sync on reconnect.
