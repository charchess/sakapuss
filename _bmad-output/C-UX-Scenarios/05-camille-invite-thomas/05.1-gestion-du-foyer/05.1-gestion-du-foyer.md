## Overview

**Page Purpose:** Manage household members — view current members, invite new ones with appropriate access profiles, and manage shared household resources (litter boxes, bowls).

**User Situation:** Camille has been using Sakapuss for a few days. She realizes Thomas contributes to cat care (cleaning litter, filling bowls) but none of his actions are tracked. She wants to add him quickly so everything gets logged.

**Success Criteria:**
- Invite flow completed in < 60 seconds
- Access profile selection is clear without documentation
- Thomas appears as "pending" immediately after invitation

**Entry Points:**
- Settings menu -> "Household members" / "Membres du foyer"
- Profile page -> "Manage household" shortcut

**Exit Points:**
- Invitation sent -> Thomas appears as "pending" in member list
- Back arrow -> previous page (settings or profile)

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Has members, no pending invites | Member list + resources + invite CTA | All actions available |
| With Pending | Invitation sent, not yet accepted | Member shows "pending" badge in `color-accent` | Can resend or revoke |
| Invite Sheet Open | User tapped invite CTA | Bottom sheet overlays page, scrim behind | Fill form + send |
| Invite Sending | Send button tapped | Button shows spinner, disabled | Wait |
| Invite Sent | API confirms delivery | Sheet closes, toast: "Invitation sent!" | Return to list |
| Invite Error | Email delivery fails | Inline error below email field | Retry |
| Empty Household | Only current user (edge case, first visit) | Single member card + prominent invite CTA | Invite first member |
| Offline | No network | Member list from cache, invite CTA disabled with tooltip | View only |

## Technical Notes

- **Permissions:** Only Admin/Total profiles can access this page. Saisie and Consultation users see a read-only version without invite CTA or resource management.
- **Email validation:** Client-side format check. Server-side: check if email already in household, show appropriate message.
- **Invitation link:** Contains household ID + profile preset. Link expires after 7 days, resendable.
- **Profile preview:** Rendered client-side from profile config, not a screenshot. Lightweight component reusing dashboard grid logic.
- **Resources:** Named resources (caisses, gamelles) are household-level, shared by all members. They appear as selection options in Quick Log flows.
- **Offline:** Member list cached. Invite requires network. Show disabled state with explanation.
