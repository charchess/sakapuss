## Overview

**Page Purpose:** Thomas receives Camille's invitation, understands his role in one sentence, and creates an account with zero friction. No complex onboarding, no tutorials, no multi-step wizards.

**User Situation:** Thomas got a link from Camille (email or message). He's curious but has zero intent to learn anything complex. He opens the link on his phone. He needs to understand what Sakapuss is and what he's supposed to do in under 10 seconds.

**Success Criteria:**
- Invitation acceptance in < 30 seconds
- Thomas understands his role without reading documentation
- Account creation via OAuth or minimal form (name + password)
- Lands on his dashboard immediately after

**Entry Points:**
- Email link: "Camille t'a ajoute a Sakapuss pour Pixel et Luna"
- Shared messaging link (WhatsApp, SMS)

**Exit Points:**
- Accept + create account -> 06.2 Dashboard Thomas
- Decline -> confirmation + link remains valid for later

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Valid invitation link opened | Welcome screen with animal photos + accept/decline | Accept or decline |
| Sheet Open | Accept tapped | Account creation bottom sheet overlays | Fill form or OAuth |
| Creating Account | Submit/OAuth in progress | Button shows spinner | Wait |
| Account Created | Success | Brief success animation -> redirect | Auto-navigate to 06.2 |
| Expired Link | Invitation > 7 days old | Error state: "This invitation has expired. Ask {inviter} to send a new one." | Close page |
| Already Accepted | Link used again after account created | Redirect to login or directly to 06.2 if authenticated | Login or auto-redirect |
| Invalid Link | Malformed or revoked | Error: "This invitation is no longer valid." | Close page |
| Form Error | Validation failure or server error | Inline error messages below relevant fields | Fix and retry |

## Technical Notes

- **Invitation token:** URL contains encrypted token with household ID, profile preset (Saisie), and invited email. Validate server-side on page load.
- **Pre-filled email:** Locked to the invitation email. If Thomas wants a different email, he must ask Camille to re-invite.
- **OAuth:** After OAuth success, server auto-links the OAuth account to the invitation token, creates user, joins household with preset profile.
- **No onboarding wizard:** Thomas sees zero tutorials, zero feature explanations. His dashboard (06.2) is self-explanatory by design.
- **Deep link:** PWA install prompt shown after account creation, not before. Don't block the flow.
- **Performance:** Page must load in < 1 second. Animal photos are pre-cached via invitation data.
