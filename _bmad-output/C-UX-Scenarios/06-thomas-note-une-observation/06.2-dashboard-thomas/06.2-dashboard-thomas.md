## Overview

**Page Purpose:** Reduced dashboard matching Thomas's "Saisie" profile -- only action tiles he can perform. No reminder management, no settings, no complexity. Thomas taps "Caisse nettoyee" -> picks "Caisse cuisine" -> done. Under 30 seconds.

**User Situation:** Thomas just joined (first visit) or opens the app as a returning user. He's standing in the kitchen after cleaning the litter box. He wants to log it and move on. He does not want to learn the app.

**Success Criteria:**
- First action logged in < 30 seconds
- Only actions matching Saisie profile visible (no confusion about what he can/can't do)
- Household resources (named litter boxes, bowls) immediately accessible
- No dead-ends or permission-denied screens

**Entry Points:**
- After onboarding (06.1) -- first visit
- PWA shortcut on home screen (returning)
- Bottom navigation "Home" tab

**Exit Points:**
- Tap quick action tile -> Quick Log flow (resource selection -> confirm)
- Tap center "+" button -> Quick Log (expanded)
- Bottom nav -> Timeline (read-only view of own entries)

## Page States

| State | When | Appearance | Actions |
|-------|------|------------|---------|
| Default | Has some activity | Grid + resources + recent entries | All Saisie actions available |
| First Visit | Just completed onboarding | Grid visible, empty recent with encouraging prompt, resources shown | Log first action |
| Loading | App opening / data sync | Skeleton grid (4 tiles) + skeleton recent | Wait |
| Offline | No network | Full UI from cache, subtle offline indicator in header | All local actions work, sync on reconnect |
| Resource Selection | Tapped a tile that needs resource choice | Bottom sheet with resource list (e.g., "Which litter box?") | Select resource -> confirm |
| Action Confirmed | Just logged an action | [Confirmation Toast](../../../D-Design-System/00-design-system.md#confirmation-toast) at bottom | Optional: add note |

## Technical Notes

- **Profile filtering:** Dashboard tiles are server-configured per profile. Saisie profile config defines which action types are available. Admin can customize this per member.
- **Resource selection flow:** When Thomas taps "Caisse nettoyee" and there are 2+ litter boxes, a minimal bottom sheet shows the named options. If only 1, skip selection -- auto-select and confirm.
- **Performance:** Dashboard must render in < 300ms. Same service worker caching as Camille's dashboard.
- **PWA:** After first visit, show "Add to home screen" prompt. Non-blocking, dismissable.
- **Activity filter:** Recent activity shows only Thomas's entries. He cannot see Camille's entries from this view (but can see all entries in Timeline if he navigates there).
- **Animal tabs:** Same behavior as Camille's dashboard. If only 1 animal, hide tabs.
- **Grid order:** Admin-configurable. Default for Saisie: Caisse, Gamelle, Observation, Evenement libre.
