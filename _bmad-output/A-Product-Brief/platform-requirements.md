## Architecture Notes

### Flux Cible

```
Téléphone (saisie) → Sakapuss API → Base de données
                                   ↓
                            Home Assistant (natif + MQTT)
                                   ↓
                      Notifications / Dashboard HA
```

### Multi-Tenant (v2)

- Isolation des données par compte (foyer)
- Portail vétérinaire = accès lecture sur subset de données patient
- Sync cloud pour self-hosters = bridge local → cloud (payant)

### Contraintes Solo Dev

- Pas de services tiers payants en phase 1
- Préférence pour self-hosted (homelab) avant cloud commercial
- Stack stable et maintenable par une seule personne
